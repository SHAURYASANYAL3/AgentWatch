### Overview
As AgentWatch transitions to include a Premium tier (AgentWatch Cloud / Enterprise), we need a seamless way for users to upgrade directly from the CLI. This requires implementing a secure CLI-to-Web payment flow, distinguishing between free and premium features, and integrating a payment gateway.

### Objectives
1. **Feature Gating**: Implement a clean separation between Open-Source/Free features and Premium features within the CLI and SDK logic.
2. **CLI Upgrade Command**: Add an `agentwatch upgrade` (or similar) command.
3. **Browser Handoff**: The CLI should generate a secure, short-lived session token and automatically open the user's default web browser to the checkout page.
4. **Payment Gateway Integration**: Integrate with a payment provider (e.g., Stripe, Paddle) on the web backend to process subscriptions.
5. **Entitlement Syncing**: Once the payment is completed, the web backend must issue a cryptographically signed license key or entitlement token that the CLI can store (e.g., in `~/.agentwatch/config.toml`) and use for subsequent premium requests.

### Acceptance Criteria
- [ ] Attempting to run a premium command on a free tier prompts the user to upgrade.
- [ ] Running `agentwatch upgrade` successfully opens the browser to the payment portal.
- [ ] Payment gateway webhooks are securely handled by the AgentWatch backend to provision access.
- [ ] The CLI successfully receives and stores the premium entitlement token post-payment.

### Security Considerations
- The session token passed from the CLI to the browser must be single-use, short-lived, and bound to the user's CLI instance to prevent interception.
- See related Anti-Tamper issue for enforcing these entitlements.
